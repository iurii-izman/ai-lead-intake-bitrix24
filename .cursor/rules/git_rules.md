# Git Rules

Source of truth: `docs/ai_lead_intake_bitrix24_tz_v1_0.md`

- Branches: `main`, `dev`, `feature/<epic-name>`.
- Commit prefixes: `docs:`, `chore:`, `feat:`, `fix:`, `test:`, `security:`, `refactor:`.
- One epic per branch.
- Preferred flow for epic work: `feature/<epic-name>` branch, commit, push, draft PR, merge after checks.
- Before merge, run the checklist and tests.
- Leave the working tree clean after each epic.
- Do not push epic work directly to `main` unless explicitly requested.
